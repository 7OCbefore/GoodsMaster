# **2025-2026 GoodsMaster Pro 演进路线图**

本文档规划了 GoodsMaster Pro 从 MVP 向大厂级工程演进的实施路径。

## **阶段一：地基加固 (Foundation) \- 2025 Q4**

**目标**: 解决最致命的性能隐患，建立类型安全网。

* **\[P0\] 数据架构迁移**:  
  * 引入 Dexie.js。  
  * 实现 LocalStorage \-\> IndexedDB 迁移脚本。  
  * 重构 useStore，剥离同步写入逻辑。  
* **\[P1\] 工程化初始化**:  
  * 配置 TypeScript 环境。  
  * 配置 Vitest 单元测试框架。  
  * 为核心 WAC 算法编写测试用例。  
* **\[P1\] 金额精度修复**:  
  * 引入 decimal.js 或整数存储方案，彻底解决浮点数隐患。

## **阶段二：体验升级 (Experience) \- 2026 Q1**

**目标**: 对齐 iOS 原生体验，提升操作效率。

* **\[P2\] 路由改造**:  
  * 引入 Vue Router。  
  * 实现页面间 Slide 切换动画。  
  * 实现 KeepAlive 缓存策略优化。  
* **\[P2\] 交互细节**:  
  * 开发 VirtualNumPad 组件。  
  * 实现列表项的 Swipe Actions (左滑删除)。  
  * 全局补充 Haptic Feedback (震动反馈)。

## **阶段三：极致性能 (Performance) \- 2026 Q2**

**目标**: 支撑万级数据量，实现秒开。

* **\[P2\] 虚拟列表**:  
  * 改造 SalesView 和 InventoryView 为虚拟滚动。  
* **\[P3\] Web Workers**:  
  * 将 Excel 导出和月度报表计算移入 Worker。  
  * 动态加载 xlsx 库。  
* **\[P3\] PWA 增强**:  
  * 完善 Service Worker 缓存策略。  
  * 优化“添加到主屏幕”引导流程。

## **阶段四：生态扩展 (Ecosystem) \- 2026 Q3+**

**目标**: 数据安全与多端协同。

* **\[P3\] 云备份 (Cloud Sync)**:  
  * 实现导出加密 JSON 到文件系统。  
  * (可选) 对接 WebDAV 或 Google Drive API 进行云端备份。  
* **\[P4\] 国际化 (i18n)**:  
  * 提取所有硬编码中文至 locales/zh-CN.json。  
  * 支持英文界面切换。

**优先级定义:**

* **P0**: 阻塞性问题或核心架构风险，必须优先解决。  
* **P1**: 严重影响开发效率或潜在 bug 风险。  
* **P2**: 重要体验提升。  
* **P3/P4**: 锦上添花的功能。
